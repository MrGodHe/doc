# 背景

![](E:\project\study-doc\doc\image\os\osFileRead.gif)

​	应用程序要读取某个文件时，并不是直接去读取文件，而是通过调用内核程序（也称 system call），内核读取文件后，再返回给应用程序。很明显同一块文件数据，在内存中保留了两份，占据了不必要的内存空间。

针对此问题，操作系统提供了内存映射机制：

- **Linux系统  mmap**
- **windows系统 Filemapping**
